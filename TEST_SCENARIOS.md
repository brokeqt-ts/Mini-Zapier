# Тестовые сценарии Mini-Zapier

Каждый сценарий — отдельный воркфлоу. Создавай через «+ Новый сценарий» на дашборде.

---

## Как вставлять переменные в шаблоны

В любом поле с надписью «+ Переменная» или `{/}` кликни кнопку — откроется список доступных переменных. Каждая переменная отображается цветной табличкой (чипом).

### Откуда берутся переменные
Каждый узел в цепочке до текущего создаёт свою группу переменных:

| Тип узла | Группа в пикере | Примеры переменных |
|---|---|---|
| TRIGGER_WEBHOOK | название узла | `trigger.message`, `trigger.headers.user-agent` |
| TRIGGER_CRON | название узла | `trigger.triggeredAt` |
| TRIGGER_EMAIL | название узла | `trigger.from`, `trigger.subject`, `trigger.body`, `trigger.date` |
| ACTION_HTTP_REQUEST | название узла | `имя_узла.status`, `имя_узла.body` |
| ACTION_DATA_TRANSFORM | название узла | `имя_узла.result.ключ` |
| ACTION_DB_QUERY | название узла | `имя_узла.rows`, `имя_узла.rowCount` |

### Важно: Webhook и поля тела запроса
Бэкенд «разворачивает» JSON-тело вебхука напрямую в `trigger.*`. Поэтому:

- Если отправил `{"message": "привет"}` → используй `{{trigger.message}}` ✅
- `{{trigger.body}}` — вернёт весь объект как JSON-строку (таблички с пометкой «целиком»)
- Если в твоём запросе поле с другим именем (например `title`) — вводи `{{trigger.title}}` вручную

### Имя узла → ключ переменной
Ключ переменной = название узла в нижнем регистре, пробелы заменены на `_`:
- `HTTP Request` → `http_request`
- `Отправить письмо` → `отправить_письмо`
- `Data Transform` → `data_transform`

## Сценарий 1 — Webhook → Telegram
**Название:** `Тест Webhook → Telegram`
**Сложность:** ⭐ простой

### Узлы
1. **TRIGGER_WEBHOOK**
   - Путь вебхука: `webhook-test`

2. **ACTION_TELEGRAM**
   - Использовать мой Telegram: ✅ (если подключён)
   - Текст сообщения — собери из табличек:
     | Текст | Переменная | Таблички в пикере |
     |---|---|---|
     `Вебхук сработал!` | — | (вводи текст вручную) |
     `От:` | `{{trigger.headers.user-agent}}` | группа триггера → «User-Agent» |
     `Тело:` | `{{trigger.message}}` | группа триггера → «Поле "message" из тела» |

   Итоговый шаблон:
   ```
   Вебхук сработал!
   От: {{trigger.headers.user-agent}}
   Тело: {{trigger.message}}
   ```

### Как тестировать

**PowerShell:**
```powershell
$bytes = [System.Text.Encoding]::UTF8.GetBytes('{"message": "привет"}')
Invoke-RestMethod -Method Post -Uri "https://mini-zapier-production-47cd.up.railway.app/api/webhooks/webhook-test" `
  -ContentType "application/json; charset=utf-8" `
  -Body $bytes
```

**bash / curl:**
```bash
curl -X POST https://mini-zapier-production-47cd.up.railway.app/api/webhooks/webhook-test \
  -H "Content-Type: application/json" \
  -d '{"message": "привет"}'
```

### Ожидаемый результат
```
Вебхук сработал!
От: Mozilla/5.0 (Windows NT …) WindowsPowerShell/5.1…
Тело: привет
```

## Сценарий 2 — Расписание → Email
**Название:** `Тест CRON → Email`
**Сложность:** ⭐ простой

### Узлы
1. **TRIGGER_CRON**
   - Расписание: выбери **«Каждые N минут»**, в поле N введи `5`

2. **ACTION_EMAIL**
   - Аккаунт: выбрать сохранённый
   - Кому: твой email для проверки
   - Тема — собери из табличек:
     | Текст | Переменная | Таблички в пикере |
     |---|---|---|
     | `Тест расписания — ` | `{{trigger.triggeredAt}}` | группа триггера → «Время запуска» |
   - Текст — собери из табличек:
     | Текст | Переменная | Таблички в пикере |
     |---|---|---|
     | `Сценарий сработал по расписанию в ` | `{{trigger.triggeredAt}}` | группа триггера → «Время запуска» |

### Как тестировать
Активируй сценарий и нажми «Тест» — сработает немедленно без ожидания расписания.

### Ожидаемый результат
Тема письма: `Тест расписания — 2026-03-18 15:49:46`
Текст: `Сценарий сработал по расписанию в 2026-03-18 15:49:46`

## Сценарий 3 — Email → Telegram

**Сложность:** ⭐⭐ средний

   - Аккаунт: выбрать сохранённый (с IMAP)

2. **ACTION_TELEGRAM**

     |---|---|

## Сценарий 4 — Webhook → Преобразование данных → Telegram
**Название:** `Тест Webhook → Transform → Telegram`
**Сложность:** ⭐⭐ средний

### Узлы
1. **TRIGGER_WEBHOOK**
   - Путь вебхука: `order-notify`

2. **ACTION_DATA_TRANSFORM** (назови узел `Data Transform`)

   Нажми **«+ Добавить поле»** три раза. Каждый блок — одно поле маппинга:

   **Блок 1:**
   - Имя поля результата: введи `orderTitle`
   - Вкладка: **Поле** (по умолчанию)
   - Источник: нажми «+ выбрать переменную» → группа триггера → табличку **`"title"`**

   **Блок 2:**
   - Имя поля результата: введи `amount`
   - Вкладка: **Поле**
   - Источник: нажми «+ выбрать переменную» → группа триггера → табличку **`"amount"`**

   **Блок 3:**
   - Имя поля результата: введи `customer`
   - Вкладка: **Поле**
   - Источник: нажми «+ выбрать переменную» → группа триггера → табличку **`"customer"`**

   Сохрани сценарий — после этого в узле Telegram появятся таблички с полями результата.

3. **ACTION_TELEGRAM**
   - Использовать мой Telegram: ✅
   - Текст сообщения — собери из табличек:
     | Текст | Переменная | Таблички в пикере |
     |---|---|---|
     | `Новый заказ!` | — | вручную |
     | `Товар:` | `{{data_transform.result.orderTitle}}` | группа «Data Transform» → **`result.orderTitle`** |
     | `Сумма:` + ` ₽` | `{{data_transform.result.amount}}` | группа «Data Transform» → **`result.amount`** |
     | `Покупатель:` | `{{data_transform.result.customer}}` | группа «Data Transform» → **`result.customer`** |

   Итоговый шаблон:
   ```
   Новый заказ!
   Товар: {{data_transform.result.orderTitle}}
   Сумма: {{data_transform.result.amount}} ₽
   Покупатель: {{data_transform.result.customer}}
   ```

### Как тестировать
**PowerShell:**
```powershell
$bytes = [System.Text.Encoding]::UTF8.GetBytes('{"title": "Ноутбук", "amount": 75000, "customer": "Иван Иванов"}')
Invoke-RestMethod -Method Post -Uri "https://mini-zapier-production-47cd.up.railway.app/api/webhooks/order-notify" `
  -ContentType "application/json; charset=utf-8" `
  -Body $bytes
```

**bash / curl:**
```bash
curl -X POST https://mini-zapier-production-47cd.up.railway.app/api/webhooks/order-notify \
  -H "Content-Type: application/json" \
  -d '{"title": "Ноутбук", "amount": 75000, "customer": "Иван Иванов"}'
```

### Ожидаемый результат
В Telegram: «Новый заказ! Товар: Ноутбук. Сумма: 75000 ₽. Покупатель: Иван Иванов»

## Сценарий 5 — Webhook → DB → Email
**Название:** `Тест Webhook → DB → Email`
**Сложность:** ⭐⭐⭐ сложный

### Предварительно — сохрани подключение к БД

1. Перейди в **Настройки → «Подключения к базам данных»**
2. Нажми **«+ Добавить подключение»**
3. Название: `Railway БД` (или любое)
4. Строка подключения: скопируй **Postgres Connection URL** из Railway (вкладка **Connect** у PostgreSQL-сервиса)
5. Сохрани

> **Где взять базу данных:** [railway.app](https://railway.app/new) · [supabase.com](https://supabase.com/dashboard/new) · [neon.tech](https://console.neon.tech/signup) — все бесплатно

---

### Шаг 1 — Создай таблицу `sync_log`

Сделай это один раз перед запуском сценария:

1. Создай **новый временный сценарий** (или используй любой существующий)
2. Добавь узел **ACTION_DB_QUERY**
3. **Подключение** → выбери `Railway БД` из выпадающего списка
4. **Режим запроса** → **«Запись»**
5. **Тип запроса** → **«CREATE TABLE IF NOT EXISTS»**
6. **Название таблицы:** `sync_log`
7. Нажми **«+ Добавить колонку»** и добавь колонки:

   | Название | Тип | NULL | DEFAULT |
   |---|---|---|---|
   | `id` | `SERIAL` | ☐ | — |
   | `workflow_id` | `TEXT` | ✓ | — |
   | `status` | `TEXT` | ✓ | — |
   | `message` | `TEXT` | ✓ | — |
   | `data` | `JSONB` | ✓ | — |
   | `created_at` | `TIMESTAMP` | ☐ | `NOW()` |

8. Нажми **«Тест»** — таблица создастся в базе
9. Таблицу нужно создать только один раз. После этого можно удалить временный сценарий.

---

### Шаг 2 — Собери основной сценарий

#### Узлы

1. **TRIGGER_WEBHOOK**
   - Путь вебхука: `sync-event`

2. **ACTION_DB_QUERY**
   - **Подключение** → выбери `Railway БД` из выпадающего списка
   - **Режим запроса:** **Запись**
   - **Тип запроса:** **INSERT — добавить запись**
   - **Таблица:** `sync_log`
   - Нажми **«+ Добавить колонку»** 4 раза:

     | Колонка | Значение | Как задать значение |
     |---|---|---|
     | `workflow_id` | `{{trigger.source}}` | таблички → **`"source"`** |
     | `status` | `ok` | просто напечатай `ok` в поле значения |
     | `message` | `{{trigger.message}}` | таблички → **`"message"`** |
     | `data` | `{}` | просто напечатай `{}` в поле значения |

   > `ok` и `{}` — статический текст, не переменные. Поле значения принимает и обычный текст, и таблички.

3. **ACTION_EMAIL**
   - **Отправить с аккаунта:** выбери Resend из списка
   - **Кому:** введи свой email
   - **Тема** — поле с кнопкой «+ Переменная»:
     1. Напечатай `Событие записано — `
     2. Нажми **«+ Переменная»** → группа триггера → **`"source"`**

     Итог: `Событие записано — {{trigger.source}}`

   - **Текст письма** — поле с кнопкой «+ Переменная»:
     1. Напечатай `Источник: `
     2. Нажми **«+ Переменная»** → группа триггера → **`"source"`**
     3. Напечатай ` Сообщение: ` (пробел до и после)
     4. Нажми **«+ Переменная»** → группа триггера → **`"message"`**

     Итог: `Источник: {{trigger.source}} Сообщение: {{trigger.message}}`

### Как тестировать
**PowerShell:**
```powershell
$bytes = [System.Text.Encoding]::UTF8.GetBytes('{"source": "crm", "message": "Новый контакт добавлен"}')
Invoke-RestMethod -Method Post -Uri "https://mini-zapier-production-47cd.up.railway.app/api/webhooks/sync-event" `
  -ContentType "application/json; charset=utf-8" `
  -Body $bytes
```

**bash / curl:**
```bash
curl -X POST https://mini-zapier-production-47cd.up.railway.app/api/webhooks/sync-event \
  -H "Content-Type: application/json" \
  -d '{"source": "crm", "message": "Новый контакт добавлен"}'
```

### Ожидаемый результат
1. Запись появилась в таблице `sync_log`
2. На email пришло уведомление

## Сценарий 6 — CRON → HTTP Request → Telegram (мониторинг)
**Название:** `Тест мониторинга сайта`
**Сложность:** ⭐⭐ средний

### Узлы
1. **TRIGGER_CRON**
   - Расписание: выбери **«Каждые N минут»**, в поле N введи `10`

2. **ACTION_HTTP_REQUEST** (назови узел `HTTP Request`)
   - URL: `https://httpbin.org/get`
   - Метод: GET

3. **ACTION_TELEGRAM**
   - Использовать мой Telegram: ✅
   - Текст сообщения — собери из табличек:
     | Переменная | Таблички в пикере |
     |---|---|
     | `{{http_request.status}}` | группа «HTTP Request» → «Код ответа (число)» |
     | `{{trigger.triggeredAt}}` | группа триггера → «Время запуска» |

   Итоговый шаблон:
   ```
   Проверка сайта ✅
   Статус: {{http_request.status}}
   Время: {{trigger.triggeredAt}}
   ```

### Как тестировать
Активируй и нажми «Тест» — сработает немедленно без ожидания.

### Ожидаемый результат
В Telegram: «Проверка сайта ✅ Статус: 200»

## Общие советы

- **Кнопка «Тест»** в редакторе запускает сценарий немедленно, не дожидаясь триггера
- **История запусков** — кнопка «История» в редакторе показывает все запуски с логами каждого узла
- **Email-триггер** опрашивает ящик раз в 60 секунд — после активации подожди минуту
- **Таблички переменных** появляются в полях с кнопкой «+ Переменная» или `{/}` — только поля до текущего узла в цепочке
- **Поля webhook-тела** вводи вручную если нужного имени нет в пикере: `{{trigger.моё_поле}}`
- **PowerShell** — используй `[System.Text.Encoding]::UTF8.GetBytes(...)` для корректной передачи кириллицы
